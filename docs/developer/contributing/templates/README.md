# Documentation Templates

Use these templates for new durable documentation. Remove instructional comments when filling a template, but preserve required headings and freshness metadata.

| Template | Use |
| --- | --- |
| [Feature documentation](feature-documentation.md) | Explain a complete user-facing or operational capability. |
| [Module documentation](module-documentation.md) | Define a module's ownership, public surface, invariants, and dependencies. |
| [ADR](adr.md) | Record a durable architectural decision. |
| [Decision log](decision-log.md) | Track smaller, local decisions that do not require an ADR. |

Templates follow the [documentation standard](../documentation-standard.md), [freshness policy](../documentation-freshness-policy.md), and [review checklist](../documentation-review-checklist.md).
