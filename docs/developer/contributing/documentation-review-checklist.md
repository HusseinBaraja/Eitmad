# راجع جودة الوثائق

اقبل التغيير فقط عندما تجيب الصفحة عن مهمة قارئ حقيقية وتطابق المصدر الحالي.

## الدليل والبنية

- [ ] كل ادعاء سلوك مدعوم بشفرة أو عقد أو مخطط أو إعداد أو اختبار أو تشغيل متحقق.
- [ ] للصفحة جمهور واحد ونمط واحد وعنوان ووصف وmetadata متطابقة.
- [ ] أقرب فهرس يربط الصفحة، ولا توجد حقيقة كنسية مكررة.
- [ ] الروابط والعناوين والأمثلة والأوامر صحيحة؛ غير المتحقق موسوم `Needs verification`.

## قابلية البحث والفعل

- [ ] تتضمن كلمات البحث العربية والاسم الهندسي الإنجليزي والمعرّفات الدقيقة ذات الصلة.
- [ ] تذكر المتطلبات والنتيجة المتوقعة وإشارات الفشل والتحقق والاسترداد.
- [ ] تقود صفحة العرض أو الخطأ إلى القدرة المالكة ومسار استرداد آمن.

## السلطة والأمان

- [ ] Rust مذكور بوصفه سلطة المجال والعقود والتخزين والصلاحيات والمزامنة والتحديث.
- [ ] الواجهة الأصلية محول رقيق ولا تنسخ مخططًا أو تحققًا أو وصول قاعدة بيانات.
- [ ] يغطي تغيير الحالة الهوية وReBAC والنطاق والتدقيق والرفض وإعادة المحاولة والفشل الجزئي والاسترداد.
- [ ] الأمثلة اصطناعية ومحددة النطاق ولا تحتوي أسرارًا أو بيانات عميل أو logs غير منقحة.

## العربية والعرض

- [ ] مساعدة المستخدم عربية أولًا وتطابق [المسرد](../../glossary.md).
- [ ] تم فحص RTL وbidi والأرقام والمعرّفات والنسخ واللصق في العرض عند انطباقها.
- [ ] الصور لها نص بديل عربي ولا تحتوي بيانات شخصية.
- [ ] بُني عارض الوثائق وفهرس البحث عند وجودهما.

## التحقق والتسليم

- [ ] نجح `audit_docs.py` واختبارات المنتج المركزة ومجموعة فحوص مساحة العمل.
- [ ] سُجلت المصادر والأوامر والمنصات التي تم فحصها والفجوات المتبقية.
- [ ] يذكر التغيير محفز المراجعة التالي.

راجع [معيار توثيق الميزات](documentation-standard.md) عندما يفشل بند.
