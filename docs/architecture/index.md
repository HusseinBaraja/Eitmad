# افهم معمارية الاعتماد

الاعتماد تطبيقات سطح مكتب أصلية رقيقة تتصل بمحرك Rust منفصل وموثوق به بوصفه سلطة المنتج. لم تُنفّذ قدرات إنتاج بعد؛ الصفحات هنا تحدد حدود الأساس المطلوب.

## ابدأ من هنا

- [المعمارية المستهدفة](target-architecture.md): شكل النظام، الملكية، IPC، التخزين، المزامنة، الأمان، التحديث، والأداء.
- [متطلبات تجربة عربية أولًا](arabic-first-ux.md): RTL، النص ثنائي الاتجاه، البحث، الإدخال، المستندات، وإمكانية الوصول.
- [القرارات المعمارية](../decisions/index.md): السجل التاريخي للاختيارات المقبولة.
- [تخطيط المستودع والملكية](../developer/repository-layout.md): أين يعيش كل حد حاليًا.

## ثوابت التغيير

- Rust يعرّف قواعد المجال والعقود والتخزين والصلاحيات والمزامنة وسياسة التحديث.
- واجهات النظام الأصلية تعرض الحالة وتتكيف مع نظام التشغيل ولا تنشئ حقيقة منتج.
- كل سجل وعملية لهما نطاق صريح، وكل تغيير حالة مخوّل ومدقق.
- كل حد خارجي مُصدّر ويستخدم تفاوض الإمكانات.
- العربية حالة التصميم والتحقق الأساسية وليست ترجمة لاحقة.

الخطوة التالية: راجع [بوابات التنفيذ الأولية](target-architecture.md#initial-implementation-gates) قبل أول قدرة رأسية.
