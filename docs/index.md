# وثائق الاعتماد | Eitmad documentation

اختر المسار الذي يطابق مهمتك. هذه الوثائق تصف الأساس الحالي فقط؛ مساحات العمل موجودة، لكن ميزات الإنتاج لم تُنفّذ بعد.

## أستخدم النظام

لا توجد إجراءات مستخدم قابلة للتوثيق بعد. عند توفر أول تدفق منتج، ستظهر مساعدة عربية في مجموعة `user/`.

## أطوّر النظام

- [ابدأ تطوير الاعتماد](developer/index.md): إعداد المساهمة، حدود الملكية، الاختبارات، وتوثيق الميزات.
- [افهم المعمارية المستهدفة](architecture/index.md): سلطة Rust، حدود الثقة، المزامنة، الأمان، والتحديثات.
- [استخدم عقود النظام](api/index.md): حالة طبقة العقود وما يجب تعريفه قبل إضافة حد خارجي.
- [راجع القرارات المعمارية](decisions/index.md): الاختيارات الدائمة وأسبابها وعواقبها.
- [استخدم المصطلحات المعتمدة](glossary.md): مفردات عربية وإنجليزية للمنتج والهندسة.

## أشغّل أو أنشر النظام

- [شغّل فحوص الأساس](operations/index.md): الأوامر الآمنة المتاحة الآن وحدود الجاهزية التشغيلية.

## ابحث حسب العرض أو المعرّف

لا توجد أخطاء منتج أو إجراءات استرداد موثقة بعد. ستُنشأ مجموعة `troubleshooting/` مع أول عرض قابل للتشخيص أو معرّف خطأ ثابت.

## سلطة الوثائق

الشفرة والاختبارات والعقود والمخططات والإعدادات الحالية هي دليل السلوك. تشرح الوثائق هذه المصادر ولا تستبدلها. عند التعارض، أصلح الصفحة الكنسية في التغيير نفسه.
